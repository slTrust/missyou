### 4-1 权限、分组与用户的关系探讨

> 权限

- API对于用户来说可不可以访问
- C端 用户 不同会员等级享受不同待遇
- CMS 动态权限组

> 分组(角色)

- 不同身份(角色)的用户可不可以访问

> #### 建议：

- 用户不和权限有关系
- 分组才和 权限有关系
- 用户必须属于一个分组

### 4-2 @ScopeLevel注解的机制

> 通过不同注解 针对一个url 设置权限

- @Admin 代表管理员
- @Vip
- @User

```

@Admin
@Vip
@User
@GetMapping("/name/{name}/with_spu")
public Theme getThemeByNameWithSpu(){ ... }
```

缺陷：

- 如果有很多接口 每个地方都要添加这个注解 管理起来非常麻烦

> @ScopeLevel

- 通过权值 来区分访问
    - VIP ==》 16
    - User ==> 8

```
@ScopeLevel(8)
@GetMapping("/name/{name}/with_spu")
public Theme getThemeByNameWithSpu(){ ... }
```
